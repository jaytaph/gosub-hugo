We're building Gosub, an ambitious project aiming to create a fully functional browser. There is a lot to do, and we welcome any help! However, as a small team, we're still working on getting fully organized. While we love to have new contributors, we sometimes struggle to clearly define tasks that can be picked up by new contributors right away.

Our first priority is to create a clear path forward. This means focussing on essential organizational tasks that we currently lack the time or energy for. We need to establish roadmaps, properly structure our documentation for new developers and streamline the onboarding process to avoid slowing down development.

In essence, we need to strike a balance between a structured organization and weekend-long hackaton. The more people who get involved, the easier it will be to create that structure.

For now, our roadmap includes:
  • Documenting how to compile and run the current Gosub components.
  • Reviewing and updating existing documentation to ensure it's accurate and complete.
  • Discussing the way forward by setting up a clear roadmap and outlining coding styles and workflows we want to follow.

Don't let this process discourage you! Even if you don't jump straight into coding like you may be able to in more mature projects, there are plenty of areas where your input will be valuable. We look forward to your contributions.
